# 美股 QDII 估值助手 · 正式第一版规格（SPEC v1）

本文件是项目唯一正式规格文档，涵盖产品、界面、算法、数据源与技术方案。

## 1. 产品定位

- 形态：Cursor / VSCode 本地扩展，个人自用，不发布 Marketplace、不建后端、不建数据库。
- 价值：查看指定 QDII 基金的估算涨跌幅、持仓贡献与历史回测误差。
- 关注基金：

| 代码 | 名称 |
|---|---|
| `270023` | 广发全球精选 |
| `501226` | 长城全球新能源车 |
| `017436` | 华宝纳斯达克精选 |

## 2. 界面结构

主界面自上而下三区：**顶部指标区 → 算法 Tab → 基金列表**。顶部不展示任何算法说明卡片。

### 2.1 顶部指标区（固定）
- 纳斯达克指数
- 纳斯达克100指数
- 标普500指数
- 美元人民币汇率
- 数据更新时间
- 刷新按钮

### 2.2 算法 Tab
- 算法1：全量持仓估值
- 算法3：快速估值

点击算法3 → 用算法3计算并展示三只基金列表；点击算法1 → 用算法1计算并展示三只基金列表。

### 2.3 基金列表（仅核心结果）
每行展示：
- 基金名称
- 基金代码
- 最新估值涨跌幅
- 估值时间
- 数据获取时间

列表不展示 alpha / beta / MAE / 方向正确率。

### 2.4 基金详情（点击列表项弹出）
- 算法3：展示前十大持仓。
- 算法1：展示最新报告能获取到的全部持仓。

每行持仓展示：持仓名称、股票代码、市场、占净值比例、当日涨跌幅、贡献值、数据来源、数据获取时间。

算法1 详情额外展示：报告截止日、报告类型、数据获取时间、全量持仓合计占比、股票类资产占比、归一化覆盖说明。

> 示例：报告截止日 2025-12-31｜持仓覆盖 70%｜股票类资产占比 79%｜剩余股票类仓位 9%，按已披露全量持仓归一化收益补足。

### 2.5 历史回测弹窗
每只基金提供「历史回测」入口，点击时实时获取并计算，展示：
- 头部指标：alpha、beta、MAE、RMSE、方向正确率、数据获取时间。
- 正文明细：每个净值日的实际净值涨跌幅、估值涨跌幅、每日误差。

## 3. 算法

内部计算统一使用小数，展示时转百分比。红涨绿跌。

### 3.1 算法3：快速估值
持仓来源：最新季度前十大。
```
前十大加权贡献   = Σ(w_i × r_i)
前十大归一化收益 = 前十大加权贡献 / 前十大权重合计
股票类资产占比   = 股票占比 + 存托凭证占比
raw估值          = 前十大归一化收益 × 股票类资产占比 + 汇率贡献
最终估值         = alpha + beta × raw估值
```

### 3.2 算法1：全量持仓估值
持仓来源（混合）：
1. 拉 2025-12-31 全量底仓。
2. 拉最新十大。
3. 用最新十大按 secid 替换全量里的同股票。
4. 按权重重新排序后计算。

```
全量持仓加权贡献   = Σ(w_i × r_i)
全量持仓归一化收益 = 全量持仓加权贡献 / 全量持仓权重合计
股票类资产贡献     = 全量持仓归一化收益 × 股票类资产占比
raw估值            = 股票类资产贡献 + 汇率贡献
最终估值           = alpha + beta × raw估值
```

算法1 与算法3 结构一致：用已披露持仓的归一化收益代理**整个**股票类仓位（含未披露剩余部分），仅数据覆盖范围更大，理论上更准。

### 3.3 alpha / beta 拟合
alpha、beta 来自历史回测自动拟合：
```
beta  = cov(raw, actual) / var(raw)
alpha = mean(actual) − beta × mean(raw)
```
若拟合后 MAE 低于原始估值则采用拟合结果。

### 3.4 误差指标（仅评估，不参与估值公式）
- MAE：平均绝对误差。
- RMSE：均方根误差。
- Bias：平均偏差。
- 方向正确率：`sign(estimate) == sign(actual)`。

## 4. 数据源

所有数据点击/刷新时实时获取，每类数据均记录**数据来源**与**数据获取时间**。

| 数据 | 用途 |
|---|---|
| 指数行情 | 顶部指标 |
| 汇率行情 | 顶部指标 + 汇率贡献 |
| 基金前十大持仓 | 算法3、算法1 混合 |
| 基金 2025-12-31 全量持仓 | 算法1 混合底仓 |
| 基金资产配置 | 股票类资产占比 |
| 股票日线行情 | 逐日估值、历史回测 |
| 基金历史净值 | 历史回测实际涨跌 |

### 4.1 顶部指标 secid（东方财富 push2）
接口：`https://push2.eastmoney.com/api/qt/ulist.np/get`，字段 `f2` 价、`f3` 涨跌幅、`f12` 代码、`f13` 市场、`f14` 名称。

| 指标 | secid |
|---|---|
| 纳斯达克指数 | `100.NDX` |
| 纳斯达克100指数 | `100.NDX100` |
| 标普500指数 | `100.SPX` |
| 美元人民币汇率 | `133.USDCNH` |

更新时间取当前北京时间（UTC+8）。

### 4.2 基金持仓接口（东方财富 fundf10）
接口：`https://fundf10.eastmoney.com/FundArchivesDatas.aspx?type=jjcc&code={code}&topline={n}&year={y}&month={m}`（GBK 编码）。
- 算法3 前十大：`topline=10&year=&month=`。
- 算法1 全量底仓：`topline=1000&year=2025&month=12`。

### 4.3 日期口径
净值日对应美股收盘估值以北京时间对齐：净值日 `T` 对应北京时间 `T+1 04:00` 的美股收盘。回测时按此口径对齐净值日与美股交易日。

## 5. 技术方案

### 5.1 架构
纯本地扩展，无后端、无数据库。
```
Extension Host（extension.ts）
  ├─ 注册命令 / Webview 承载
  └─ 取数子进程（scripts/live-data.js）：持仓、行情、指标、汇率、估值、回测
Webview（media/main.js + main.css）
  └─ 指标区 / 算法 Tab / 基金列表 / 详情弹窗 / 历史回测弹窗
```

### 5.2 关键文件
| 文件 | 作用 |
|---|---|
| `src/extension.ts` | 激活、注册 Webview、命令 |
| `src/webview/QdiiViewProvider.ts` | Webview 承载、getHtml、refresh 取数并推送 |
| `scripts/live-data.js` | 取数与估值计算（子进程） |
| `media/main.js` / `main.css` | 前端渲染 |
| `data/backtest-20d.json` | 算法3 偏移量参考样本 |

### 5.3 命令
| 命令 | 说明 |
|---|---|
| `meiguguzhi.open` | 打开估值界面 |
| `meiguguzhi.refresh` | 刷新当前估值 |
| `meiguguzhi.backtest` | 历史回测 |

### 5.4 数据模型
```ts
interface Holding {
  secid: string;
  code: string;
  name: string;
  market: 'US' | 'HK' | 'CN';
  weightPct: number;
  returnPct: number | null;
  contributionPct: number | null;
  source: string;
}

interface Valuation {
  fundCode: string;
  fundName: string;
  mode: 'algorithm1' | 'algorithm3';
  tradeDate: string;
  fxPct: number;
  rawEstimatePct: number;
  tunedEstimatePct: number;
  holdings: Holding[];
  dataFetchedAt: string;
}
```

### 5.5 错误处理
- 网络请求失败自动重试（指数退避）。
- 单只基金估值失败不影响其他基金。
- 详细错误写入 Output Channel。

## 6. 非目标

不做：用户登录、云端数据库、小程序发布、自动交易或投资建议、预测最终净值、多账户持仓管理。

## 7. 风险提示

界面必须标注：场外基金没有真正实时行情，当前结果为基于公开持仓与市场行情的算法估算，不代表基金公司最终披露净值。
