# 数据源与算法

## 基本事实

场外 QDII 基金没有真正的实时成交价格。基金公司最终披露的是每日净值，通常在下一个交易日更新。支付宝、天天基金、银行 App 或养基宝中看到的“实时估值”，本质都是算法估算。

本项目当前以三个信息源组合估值：

- 基金公开持仓：前十大、年报/半年报披露的更多权益持仓。
- 市场行情：美股、港股、A 股个股或指数的收盘涨跌。
- 汇率：美元人民币变化对人民币计价 QDII 的影响。
- 历史净值：用于计算 alpha、beta、MAE、RMSE 和方向命中率。

## 当前数据文件

`data/backtest-20d.json` 仅作为算法 3 的历史校准和回测样本。它不是完整行情库，也不是算法 1 的历史依据。

`data/algorithm1-latest.json` 是算法 1 的当前披露持仓快照，用于展示和最新估算。后续应改成刷新时自动重新抓取，并记录数据获取时间、报告期和来源。

## 日期口径

用户确认的口径：

- 净值日 `2026-07-01` 对应北京时间 `2026-07-02 04:00` 的美股收盘估值。
- 净值日 `2026-07-02` 对应北京时间 `2026-07-03 04:00` 的美股收盘估值。

因此回测时，净值日与对应美股交易日收盘涨跌要按这个北京时间口径对齐。

## 算法 3：快速估值 + 历史校准

这是当前主算法。

输入：

```text
w_i       = 第 i 只前十大持仓占基金净值比例
r_i       = 第 i 只持仓在目标交易日的涨跌幅
W_top     = 前十大可用持仓权重之和
W_equity  = 股票类仓位，优先取股票 + 存托凭证
R_fx      = 美元人民币汇率贡献
alpha     = 历史校准截距
beta      = 历史校准斜率
```

步骤：

```text
R_top_known  = sum(w_i * r_i)
R_stock_style = R_top_known / W_top
R_raw = R_stock_style * W_equity + R_fx
R_final = alpha + beta * R_raw
```

中文解释：

前十大如果合计占 40%，这 40% 当天整体表现为 -3%，就假设剩余股票仓位也大致是 -3%。然后不是直接把基金算成 -3%，而是再乘以基金真实的股票类仓位。例如股票类仓位 90%，汇率贡献 -0.06%，则原始估值为：

```text
R_raw = -3% * 90% + -0.06% = -2.76%
```

最后用历史实际净值做线性校准：

```text
actual ~= alpha + beta * raw
```

```text
beta = covariance(raw, actual) / variance(raw)
alpha = mean(actual) - beta * mean(raw)
```

如果校准后的 MAE 比原始估值更低，就使用校准结果。alpha 主要修正长期偏移，beta 主要修正估值波动偏大或偏小。

## 算法 1：全量持仓估值

算法 1 和算法 3 的思想一致，区别是算法 1 使用“最新财报能获取到的全部披露持仓”，而不是只使用前十大。

输入：

```text
w_i       = 第 i 只披露持仓占基金净值比例
r_i       = 第 i 只披露持仓涨跌幅
W_known   = 可用披露持仓权重之和
W_equity  = 股票类仓位，优先取股票 + 存托凭证
R_fx      = 汇率贡献
```

步骤：

```text
R_known = sum(w_i * r_i)
R_style = R_known / W_known
W_unknown = max(W_equity - W_known, 0)
R_unknown = R_style * W_unknown
R_raw = R_known + R_unknown + R_fx
R_final = alpha + beta * R_raw
```

中文解释：

如果财报披露了 30 个持仓，合计占基金净值 70%，而基金股票类仓位是 79%，就先按真实权重计算这 70% 的贡献，再把剩下 9% 视作和已披露持仓风格接近。这样比前十大归一化更完整，但取决于财报持仓的新旧程度。

## 算法 2：板块补偿，暂不启用

设计目标：当全量持仓不可用或过旧时，用板块代理补偿未披露部分。例如：

```text
R_known = sum(w_i * r_i)
W_unknown = max(W_equity - W_known, 0)
R_sector = sum(sector_weight_j * sector_return_j)
R_unknown = W_unknown * R_sector
R_raw = R_known + R_unknown + R_fx
R_final = alpha + beta * R_raw
```

板块来源不建议自行猜测，应该来自公开数据中的行业分类、基金报告行业暴露或可信的第三方接口。当前没有稳定来源，所以页面暂时隐藏算法 2。

## 误差指标

MAE、RMSE 和方向命中率只用于评估算法，不是估值公式本身。

- MAE：平均绝对误差，越小越好。
- RMSE：均方根误差，对大误差更敏感。
- Bias：平均偏差，用来观察系统性高估或低估。
- 方向命中率：估值涨跌方向是否与实际净值涨跌一致。
