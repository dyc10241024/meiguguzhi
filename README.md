# 美股 QDII 估值助手

这是一个个人自用的 Cursor / VSCode 扩展，用来查看美股 QDII 场外基金的估算涨跌幅和历史回测。

场外 QDII 基金没有真正的盘中实时净值。这里展示的是“估值”：用公开持仓、股票收盘涨跌、美元人民币汇率和历史净值误差校准，估算下一个净值披露前可能接近的涨跌幅。

## 当前页面

扩展左侧视图包含：

- 顶部指标：纳斯达克指数、纳斯达克 100、标普 500、美元人民币汇率、数据更新时间。
- 算法 tab：算法 3 快速估值、算法 1 全量持仓。
- 基金列表：广发全球精选 `270023`、长城全球新能源车 `501226`、华宝纳斯达克精选 `017436`。
- 详情弹窗：算法 3 展示前十持仓，算法 1 展示当前已抓取的全部披露持仓。
- 历史曲线弹窗：展示历史实际涨跌、估值涨跌、误差，以及 alpha、beta、MAE、RMSE、方向命中率。

算法 2“板块补偿”暂时隐藏，后续有可靠板块数据源后再启用。

## 三种算法的中文解释

### 算法 1：全量持仓

使用年报或半年报里能拿到的全部权益持仓。比如披露了 30 个持仓，合计占基金净值 70%，而最新资产配置里股票类仓位是 79%，那么先用这 70% 的持仓计算组合涨跌，再把剩下 9% 按同一批披露持仓的整体风格补齐。最后叠加汇率影响。

它比只看前十大更完整，但缺点是全量持仓通常来自年报/半年报，可能比季度前十大更旧。

### 算法 2：板块补偿

暂未在页面展示。设计思路是：如果只能拿到部分持仓，就用公开数据里的行业/板块暴露，再用 ETF 或指数代理未披露部分。例如半导体用 `SMH` 或 `SOXX`，科技用 `XLK`。

这一层需要可靠的板块识别数据源，目前先不启用。

### 算法 3：快速估值

这是当前主算法。它假设前十大重仓股能代表基金整体风格。

做法是：先计算前十大持仓的加权涨跌，并按前十大权重归一化。也就是说，假设前十大占 40%，这 40% 当天整体跌了 3%，那么未披露的剩余股票仓位也按接近 3% 处理。然后乘以基金最新股票类仓位，加上美元人民币汇率影响，最后用历史净值做 alpha/beta 校准。

alpha 用来修正常见偏移，beta 用来修正估值波动过大或过小。MAE 和 RMSE 不是估值公式的一部分，它们是用来衡量历史误差大小的指标。

## 开发命令

```bash
npm install
npm run compile
```

在 Cursor / VSCode 中调试：

1. 打开 `D:\meiguguzhi`。
2. 执行 `npm run compile`。
3. 按 `F5` 启动扩展开发宿主。
4. 在新窗口左侧打开“美股估值 / QDII 估值”。

## 数据文件

- `data/backtest-20d.json`：算法 3 的 20 个净值日回测和校准样本。
- `data/algorithm1-latest.json`：算法 1 当前抓取到的披露持仓和最新估算。
- `docs/data-and-algorithm.md`：数据源、公式和后续补充规则。
